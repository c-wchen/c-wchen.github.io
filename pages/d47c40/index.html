# 面向切面编程（Aspect Oriented Programming）

通过预编译方式和运行期动态代理实现程序功能的统一维护的一种技术。主要使用在日志记录，性能统计，安全控制等场景。

# 基本概念

# 1.1 通知（Advice）

前置通知(Before)：在目标方法被调用之前调用通知功能(应用的校验)

后置通知(After)：在目标方法完成之后调用通知，此时不关心方法的输出结果是什么（清理现场）

返回通知(After-returning)：在目标方法成功执行之后调用通知（常规数据处理）

After和After-returning的区别

After切入点目标方法抛出异常也会执行，After-returning必须正常返回之后才会执行

异常通知(After-throwing)：在目标方法抛出异常后调用通知（包装异常信息）

环绕通知(Around)：通知包裹了被通知的方法，在被通知的方法调用之前和调用之后执行自定义的行为（可以做任何事情）

# 1.2 连接点(Join point)

连接点是在应用执行过程中能够插入切面的一个点，这个点可以是调用方法时、抛出异常时、修改某个字段时。

类中的所有方法

# 1.3 切点(Pointcut)

切点是为了缩小切面所通知的连接点的范围，即切面在何处执行。我们通常使用明确的类和方法名称，或者利用正则表达式定义所匹配的类和方法名称来指定切点。

Join Point和Pointcut的区别

在 Spring AOP 中, 所有的方法执行都是 Join point. 而 point cut 是一个描述信息, 它修饰的是 Join point, 通过 point cut, 我们就可以确定哪些 Join point 可以被织入 Advice. 因此 join point 和 point cut 本质上就是两个不同纬度上的东西.

# 1.4 切面(Aspect)

切面是通知和切点的结合。通知和切点共同定义了切面的全部内容：它是什么，在何时和何处完成其功能。（通知与切入点关系集合）

# 1.5 引入(Introduction)

引入允许我们在不修改现有类的基础上，向现有类添加新方法或属性。

public interface FlyService {
    void fly();
}

@Component
public class FlyServiceImpl  extends DelegatingIntroductionInterceptor implements FlyService {
    public void fly() {
        System.out.println("==============fly============");
    }
}

public interface PersonService {
    void run();
}

@Component
public class PersonServiceImpl implements PersonService {
    public void run() {
        System.out.println("=============== run =============");
    }
}

通过xml的形式把FlyService方法引入PersonServiceImpl

<context:annotation-config></context:annotation-config>

# 1.6 织入(Weaving)

织入是把切面应用到目标对象并创建新的代理对象的过程。

切面在指定的连接点被织入到目标对象中，在目标对象的生命周期里，有以下几个点可以进行织入：

  • 编译期：切面在目标类编译时被织入。这种方式需要特殊的编译器。AspectJ的织入编译器就是以这种方式织入切面的。
  • 类加载期：切面在目标类加载到JVM时被织入。这种方式需要特殊的类加载器(ClassLoader)，它可以在目标类被引入应用之前增强该目标类的字节码。
  • 运行期：切面在应用运行的某个时刻被织入。一般情况下，在织入切面时，AOP容器会为目标对象动态地创建一个代理对象。Spring AOP就是以这种方式织入切面的。

通配符使用

4. * *.meth*(..)

6. * *.method1(int) //匹配int类型参数
7. * *.method1(*, int)
8. * *.method1(int, ..)

    </aop:aspect>
</aop:config>

# 使用注解方式

@Aspect
@Component
public class Log {

        System.out.println(Arrays.toString(joinPoint.getArgs()));
        System.out.println("======================= before =======================");
    }
    @After("execution(* com.wchen.service.OrderService.updateOrder(String))")
    public void after(JoinPoint joinPoint) {
        System.out.println(Arrays.toString(joinPoint.getArgs()));
        System.out.println("======================= before =======================");
    }
}

# 多种切入点方式

  1. execution

    最为常用，有多种定制化使用

  2. args

    包含指定参数类型

  3. target

    目标对象为指定对象时被切入

  4. this

    被代理切入之后为指定类型时被切入

  5. @annotation

    通过在方法使用注解的形式进行寻找切入点

    @Retention(RetentionPolicy.CLASS)
@Target(ElementType.METHOD)
public @interface AopAnnotation {
}

// ....
@Component
public class OrderServiceImpl {
    @AopAnnotation
    public void updateOrder() {
        System.out.println("update order");
    }
    public void queryAll() {
        System.out.println("find all order");
    }
}

//....
public class Log {
    // 从使用该注解的方法进行代理
    @Pointcut("@annotation(com.wchen.aop.AopAnnotation))")
    public void comm() {}

    @After("comm()")
    public void afterLog() {
        log.info("===after log===");
    }

}

  6. @within

    指定包下的所有之类或者类上有注解

    @within("com.wchen.service")

  7. @args

    参数包含指定注解

  8. @target

    判断目标对象是否包含注解

  9. 逻辑模式

    execution(* com.wchen..(..)) && args(name, age)

# 参数获取

  1. 使用JoinPoint
@Around("execution(* com.wchen.controller.TestController.login(..))")
public Object around(ProceedingJoinPoint joinPoint) throws Throwable {
    String userName = (String) joinPoint.getArgs()[0];
    String password = (String) joinPoint.getArgs()[1];
    if (!("admin".equals(userName) && "123".equals(password))) {
        log.info("login not successful");
        return "not successful";
    }
    return joinPoint.proceed();
}
  1. 使用args
@Before("execution(* com.wchen.controller.TestController.login(..)) && args(userName, password)")
public void before(String userName, String password) {
    log.info("=== from controller layer args userName = {}, password = {} ===", userName, password);
}
  1. 处理返回值
@AfterReturning("execution(* com.wchen.controller.TestController.login(..))", returning="result")
public void afterReturning(Object result) {
    //...
}
  1. 处理异常
@AfterThrowing("execution(* com.wchen.controller.TestController.login(..))", throwing="e")
public void afterReturning(Throwable e) {
    //...
}

# 动态代理

  1. JDK动态代理，实现接口或接口进行操作并不能对类直接操作

    对对象进行代理

public interface OrderService {
    void hello();
    void greeting();
}

/**
 * @author  wchen
 * @since 2020/11/29
 * 单独对接口进行实现, 区别与对代理对象的增强
 */
public class OrderProxy {
    private static class OrderMapperProxy implements InvocationHandler {

        /**
         *
         * @param proxy   被代理的对象
         * @param method  目标对象的方法对象
         * @param args    该方法的参数列表
         * @return
         * @throws Throwable
         */
        public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
            System.out.println(method.getName() + " is implements by proxy");
            System.out.println(method.getName() + " returning");
            return null;
        }
    }
    @SuppressWarnings("all")
    public static<T> T createProxyInstance(Class<T> mapper) {
        /**
         *
         * 参数列表
         * 1. 类加载器 ClassLoader loader
         * 获取方式
         *      * Thread.currentThread().getContextClassLoader();
         *      * mapper.getClassLoader();
         * 2. 待实现的接口集合 Class<?>[] interfaces
         *      * mapper.getClass().getInterfaces()
         * 3. 实现InvocationHandler的对象
         */

        return (T) Proxy.newProxyInstance(
                mapper.getClassLoader(),
                new Class[] { mapper },
                new OrderMapperProxy()
        );
    }
}

  1. cglib（code generate library）

    对类字节码进行代理, 实现子类继承的实现方式动态生成被代理类的子类

    1. 生成字节码对象
    2. 设置字节码的父类的目标对象
    3. 通过生成的字节码对象进行回调方法，在回调的过程中追加代码
    4. 创建代理对象
    5. 调用代理对象的方法，方法功能就增强
public class OrderServiceImpl implements OrderService {
    public void hello() {
        System.out.println("order service");
    }

    public void greeting() {

    }
}

public class OrderImplProxy {
    /**
     * 1. 生成字节码对象
     * 2. 设置父类对象
     * 3. 被增强的方法、回调方法
     */
    @SuppressWarnings("all")
    public static<T>  T  createInstance(T obj) {
        Enhancer enhancer = new Enhancer();
        enhancer.setSuperclass(obj.getClass());
        Callback callback = new CallbackMethod();
        enhancer.setCallback(callback);
        return (T) enhancer.create();
    }

    private static class CallbackMethod implements MethodInterceptor {
        /**
         *
         * @param o            代理对象
         * @param method       目标对象中的方法
         * @param objects      目标对象的参数
         * @param methodProxy  代理对象中代理方法对象
         * @return
         * @throws Throwable
         */
        public Object intercept(Object proxy, Method method, Object[] args, MethodProxy methodProxy) throws Throwable {
            // 不能用invoke
            System.out.println(method.getName() + " implements by proxy");
            methodProxy.invokeSuper(proxy, args);
            System.out.println("returning");
            return null;
        }
    }
}
