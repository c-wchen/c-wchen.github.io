# 3. 根文件系统

内核启动， boot loader 加载完内核文件 vmlinuz 后，紧接着挂载磁盘根文件系统，但如果此时内核没有相应驱动，无法识别磁盘，就需要先加载驱动，而驱动又位于 /lib/modules，得挂载根文件系统才能读取，这就陷入了一个两难境地，系统无法顺利启动。于是有了 initramfs 根文件系统，其中包含必要的设备驱动和工具，boot loader加载 initramfs 到内存中，内核会将其挂载到根目录 /，然后运行 /init 脚本，挂载真正的磁盘根文件系统。

下面采用两种方法实现简单方法挂载根文件系统, 任选一种调试即可。

# 1.1 busybox

下载 busybox 源代码，解压，假设解压到 ~/busybox-1.31.0，解压完成后，跟内核一样先配置编译选项：

安装依赖

yum install glibc glibc-devel pcre pcre-devel openssl openssl-devel systemd-devel zlib-devel libmcrypt-devel glibc-static ncurses-devel -y

1. 配置
cd ~/busybox
make menuconfig

2. 选择配置项
Settings  --->
    [*] Build static binary (no shared libs)
3. 编译
make -j$(nproc) && make install  ## noproc命令可查看CPU内核数

4. 在 ~/busybox目录下执行以下命令
mkdir initramfs
cd initramfs
cp ../_install/* -rf ./
mkdir dev proc sys
mkdir -p lib/modules
mkdir -p etc/init.d
touch etc/init.d/rcS
chmod a+x etc/init.d/rcS
sudo cp -a /dev/{null,console,tty,tty1,tty2,tty3,tty4} dev/
cp ~/linux-5.3.7/drivers/net/ethernet/intel/e1000/e1000.ko lib/modules
rm linuxrc
vim init
chmod a+x init

vim init

#!/bin/sh
mount -t proc none /proc
mount -t sysfs none /sys
insmod /lib/modules/e1000.ko
/sbin/mdev -s
setsid cttyhack /bin/sh
exec /sbin/init

打包initramfs

find . -print0 | cpio --null -ov --format=newc | gzip -9 > ../initramfs.cpio.gz

启动内核并调试

qemu-system-x86_64 -s -S -kernel ~/linux-5.3.7/arch/x86/boot/bzImage -initrd ~/busybox-1.31.0/initramfs.cpio.gz -nographic -append "nokaslr console=ttyS0" -serial mon:stdio -device e1000,netdev=net0 -netdev user,id=net0,hostfwd=tcp::5555-:22

启动调试

gdb vmlinux

(gdb) target remote :1234
(gdb) b start_kernel
(gdb) c

sudo qemu-system-x86_64 \
-kernel ./arch/x86/boot/bzImage \
-initrd ./rootfs \

Qemu 开启GDB调试

sudo qemu-system-x86_64  \
-kernel ./arch/x86/boot/bzImage  \
-initrd ./rootfs  \
-append "nokaslr root=/dev/ram rdinit=/debug_kernel" \
-smp 2  \
-s -S

进行以上会打开Qemu并进入等待调试状态，此时可以直接gdb调试，如下（非必须）

# 4. 错误总结

  1. 内核错误: No rule to make target ‘debian/canonical-certs.pem‘, needed by ‘certs/x509_certificate_list‘

修改.config文件CONFIG_SYSTEM_TRUSTED_KEYS，将其置空。

vim .config
# 将该项原有内容删掉即可，如下
CONFIG_SYSTEM_TRUSTED_KEYS=""
  1. 打断点执行无法停止（一直continuing...）

内核启动参数上添加 nokaslr： 关闭KASLR技术

KASLR技术让kernel image映射的地址相对链接地址偏移。偏移地址可以通过dts设置。如果bootloader支持每次开机随机生成偏移数值，那么可以做到每次开机kernel image映射的虚拟地址都不一样。因此，对于开启KASLR的kernel来说，不同的产品的kernel image映射的地址几乎都不一样。因此在安全性上有一定的提升。

基于Nokaslr的Linux内核编译与单步跟踪 (opens new window)

  1. rpm包devel包

不带devel后缀的package，通常只包含能让程序运行的动态库和配置文件。而带devel后缀的package，则包含使用这个package开发程序的所有的必需文件。比如头文件，等等。有时devel package还包含静态库。
