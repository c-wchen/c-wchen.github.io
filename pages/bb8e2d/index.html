在开发种类型转换是普遍的，编程规范要求隐式转换改为显示转换数据类型，使其运算符合输出预期，

# 隐式转换

如果运算符两边类型不同，先将类型转换为相同的类型，即较低的类型转换为较高的类型，然后进行运算。

转换规则如下：

横向必须先转换才能运算；纵向高低优先级，需要先统一转换为高优先级再进行运算。

  • 在混合运算过程种，编译器自动转换操作符两边为相同数据类型进行运算

  • char 和 short 参与运算时，先转换成 int 类型。

  • 所有的浮点运算都是以双精度进行的，即使运算中只有 float 类型，也要先转换为 double 类型，才能进行运算。

  • 赋值运算中，赋值号两边的数据类型不同时，需要把右边表达式的类型转换为左边变量的类型

数据类型 数据类型 运算后结果类型
char-->int char-->int int
char-->int unsigned char-->int int
short-->int short-->int int
int-->unsigned int unsigned int unsigned int
float --> double float --> double double
float -->double double double
int-->long long long

表格种两种数据类型表示运算符左右两边的类型，符号**-->**表示先转换到指向类型再运算。

# 强制转换

自动转换不一定安全，可能出现意料之外的结果，所以在编码种需要强制转换一下类型，保障数据不失真、精度降低、回绕、溢出等问题

  1. 临时性

无论是自动类型转换还是强制类型转换，都只是为了本次运算而进行的临时性转换，转换的结果也会保存到临时的内存空间，不会改变数据本来的类型或者值。

  1. 符号位扩展
转换前 转换后 低类型转高类型 高类型转低类型
signed signed 直接赋值，符号位扩展 截断高字节
signed unsigned 直接赋值，符号位扩展 截断高字节
unsigned unsigned 直接赋值，填零扩展 截断高字节
unsigned signed 直接赋值，填零扩展 截断高字节

# 转换：short ->int转换

#include <stdio.h>

int main() {
    // signed -> signed, unsigned
    short             a = 0xF1F1;
    int               b = a;
    unsigned int      c = a;
    
    // unsigned -> signed, unsigned
    unsigned short    d = 0xF1F1;
    int               e = d;
    unsigned int      f = d;
    printf("a = %hx, b = %x, c = %x, d = %hx, e = %x, f = %x\n", a, b, c, d, e, f);
}


// output==>
// a = f1f1, b = fffff1f1, c = fffff1f1, d = f1f1, e = f1f1, f = f1f1

# 转换：int -->long long

#include <stdio.h>

int main() {
    // signed -> signed, unsigned
    int                         a = 0xF1F1F1F1;
    long long int               b = a;
    unsigned long long int      c = a;
    
    // unsigned -> signed, unsigned
    unsigned int                d = 0xF1F1F1F1;
    long long int               e = d;
    unsigned long long int      f = d;
    printf("a = %x, b = %llx, c = %llx, d = %x, e = %llx, f = %llx\n", a, b, c, d, e, f);
}

// output ==>
// a = f1f1f1f1, b = fffffffff1f1f1f1, c = fffffffff1f1f1f1, d = f1f1f1f1, e = f1f1f1f1, f = f1f1f1f1

# 转换：int --> short

#include <stdio.h>

int main() {
    // signed -> signed, unsigned
    int                 a = 0xF8F7F6F5;
    short               b = a;
    unsigned short      c = a;
    
    // unsigned -> signed, unsigned
    unsigned int        d = 0xF8F7F6F5;
    short               e = d;
    unsigned short      f = d;
    printf("a = %x, b = %hx, c = %hx, d = %x, e = %hx, f = %hx\n", a, b, c, d, e, f);
}
// output=>
// a = f8f7f6f5, b = f6f5, c = f6f5, d = f8f7f6f5, e = f6f5, f = f6f5

总结一下: 有符号转高类型数据，符号位扩展；无符号转高类型数据，填零扩展

# 溢出和回绕

有符号溢出（与编译器实现有关，对于大多数编译器来说处理方式还是回绕），无符号回绕

# 可能危害

  1. 整数回绕之后，会导致索引越界，取到不确定的数据

  2. 或者判断失效，形成死循环。

  3. 回绕之后，导致分配超大内存。

  4. 拒绝服务
