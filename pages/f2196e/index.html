# jQuery.ajax（url [，settings])

  • url

    类型：String

    包含请求发送到的URL的字符串。

  • setting

    类型：plainObject

    一组配置Ajax请求的键/值对。所有设置都是可选的。可以使用$ .ajaxSetup()为任何选项设置默认值

# setting

  • headers（默认：{}）

    类型：PlainObject

    使用XMLHttpRequest传输与请求一起发送的附加标头键/值对的对象。标头X-Requested-With: XMLHttpRequest始终会添加，但是XMLHttpRequest可以在此处更改其默认值。headers设置中的值也可以从beforeSend函数中覆盖。

  • method

    类型：String

    HTTP方法要使用的请求（例如"POST"，"GET"，"PUT"）

  • data

    类型：PlainObject或String或Array

    数据要发送到服务器。如果还不是字符串，则将其转换为查询字符串。它被附加到GET请求的URL上。对象必须是键/值对。如果value是一个Array，则jQuery将根据traditional设置的值使用相同的键序列化多个值（如下所述）。

  • contentType（默认：'application/x-www-form-urlencoded; charset=UTF-8'）

    类型：Boolean或String

    将数据发送到服务器时，请使用此内容类型。默认值为“ application / x-www-form-urlencoded； charset = UTF-8”，在大多数情况下都可以。如果您将内容类型显式传递给$.ajax()，则该内容类型将始终发送到服务器（即使没有发送数据）

    注意： W3C XMLHttpRequest规范规定该字符集始终为UTF-8。指定另一个字符集不会强制浏览器更改编码。**注：**对于跨域请求，内容类型设置为比其他任何东西application/x-www-form-urlencoded，multipart/form-data或text/plain将触发浏览器发送预检OPTIONS请求到服务器。

  • async（默认值：true）

    类型：Boolean

    默认情况下，所有请求都是异步发送的（即true默认设置为）。如果需要同步请求，请将此选项设置为false。跨域请求和dataType: "jsonp"请求不支持同步操作。请注意，同步请求可能会暂时锁定浏览器，从而在请求处于活动状态时禁用任何操作。

  • beforeSend

    类型：Function

    请求前回调函数，可用于在发送之前对其进行操作。使用它来设置自定义标头等。返回值false将取消请求

  • success

    类型： Function(data, String textStatus, jqXHR jqXHR )

    请求成功时要调用的函数。该函数传递了三个参数：服务器返回的数据，根据dataType参数或dataFilter回调函数（如果指定）进行格式化；描述状态的字符串；和jqXHR（在jQuery 1.4.x中，XMLHttpRequest）对象。

  • errror

    类型：Function( jqXHR jqXHR, String textStatus, String errorThrown )

    请求失败时要调用的函数。该函数接收三个参数：jqXHR对象（在jQuery 1.4.x中为XMLHttpRequest），一个描述错误类型的字符串，以及一个可选的异常对象（如果发生）。对于第二个参数（除了可能的值null）是"timeout"，"error"，"abort"，和"parsererror"。发生HTTP错误时，errorThrown接收HTTP状态的文本部分，例如“未找到”或“内部服务器错误”。（在HTTP / 2中，它可能改为为空字符串）从jQuery 1.5开始，该error设置可以接受函数数组。每个函数将依次调用。 注意： *对于跨域脚本和跨域JSONP请求，不会调用此处理程序。*这是一[Ajax事件。

  • complete

    类型：Function([jqXHR，String textStatus）

    请求完成时（执行success和error）将要调用的函数

  • timeout

    类型：number

    设置请求的超时（以毫秒为单位）。值为0表示将没有超时。这将覆盖$ .ajaxSetup（） (opens new window)设置的所有全局超时。超时时间始于$.ajax拨打电话的时间；如果其他几个请求正在进行中，并且浏览器没有可用的连接，则请求可能会超时，然后才能发送。

  • cache（default：true, false for dataType 'script' and 'jsonp'）

    类型：Booleanean

    如果设置为false，将强制浏览器不缓存请求的页面。

    **注意：**设置cache为false只能与HEAD和GET请求一起正常使用。它通过在GET参数后面附加“ _ = {timestamp}”来工作。对于其他类型的请求，不需要此参数，但在IE8中，当对GET已经请求的URL进行POST时，则不需要该参数。

  • contents

    类型：PlainObject

    字符串/正则表达式对的对象，用于确定jQuery在给定内容类型的情况下如何解析响应。

  • context

    类型：PlainObject

    该对象将是所有与Ajax相关的回调的上下文。默认情况下，上下文是一个对象，代表调用中使用的Ajax设置（$.ajaxSettings与传递给的设置合并$.ajax）。例如，将DOM元素指定为上下文将使该上下文complete成为请求的回调，如下所示：

    $.ajax({ url: "test.html", context: document.body}).done(Function() { $( this ).addClass( "done" );});

  • converters（默认值：{"* text": window.String, "text html": true, "text json": jQuery.parseJSON, "text xml": jQuery.parseXML}）

    类型：PlainObject

    包含dataType到dataType转换器的对象。每个转换器的值都是一个函数，它返回响应的转换值。（添加的版本：1.5 (opens new window)）

  • accepts（default:：depends on dataType）

    类型：PlainObject

    一组键/值对，将给定映射dataType到其MIME类型，该MIME类型在Accept请求标头中发送。此标头告诉服务器它将接受什么样的MIME类型作为响应。例如，以下定义了mycustomtype与请求一起发送的自定义类型：

    $.ajax({
  accepts: {
    mycustomtype: 'application/x-some-custom-type'
  }，
  converters: {
    'text mycustomtype': Function(result) {
      return newresult;
    }
  },
 
  // 期望服务器返回值类型为mycustomtype
  dataType: 'mycustomtype'
});

    **注意：**您将需要为此类型指定一个补充条目，converters以使其正常工作。

  • crossDomain（默认值：false for same-domain requests, true for cross-domain requests）

    类型：Boolean

    如果您希望在同一域上强制执行crossDomain请求（例如JSONP），请将crossDomain的值设置为true。例如，这允许将服务器端重定向到另一个域

  • dataFilter

    类型： Function(data, String type)=> Anything

    用于处理XMLHttpRequest的原始响应数据的函数。这是一个预过滤功能，用于清理响应。您应该返回经过清理的数据。该函数接受两个参数：服务器返回的原始数据和'dataType'参数。

  • dataType（default：Intelligent Guess (xml, json, script, or html)）

    类型：String

    您期望从服务器返回的数据类型。如果未指定任何内容，则jQuery将尝试根据响应的MIME类型来推断它（XML MIME类型将产生XML，在1.4中，JSON将产生JavaScript对象，在1.4脚本中，脚本将执行该脚本，而其他任何内容将是以字符串形式返回）。可用的类型（以及作为第一个参数传递给您的成功回调的结果）是：

    • "xml"：返回可以通过jQuery处理的XML文档。
    • "html"：以纯文本形式返回HTML；插入到DOM中时，将评估包含的脚本标签。
    • "script"：将响应评估为JavaScript并以纯文本形式返回。_=[TIMESTAMP]除非将cache选项设置为，否则通过在URL 后面附加查询字符串参数来禁用缓存true。**注意：**对于远程域请求，这会将POST转换为GET。在jQuery 3.5.0之前，带有脚本的HTTP响应Content-Type仍未执行。
    • "json"：将响应评估为JSON并返回一个JavaScript对象。除非请求包含在其请求选项中，否则使用JSONP (opens new window)来执行"json"具有回调占位符的跨域请求。JSON数据以严格的方式进行解析；任何格式错误的JSON都会被拒绝，并引发解析错误。从jQuery 1.9开始，空响应也被拒绝；服务器应返回或的响应。（有关正确的JSON格式的更多信息，请参见json.org (opens new window)。）?callback=?``jsonp: false``null``{}
    • "jsonp"：使用JSONP (opens new window)加载JSON块。"?callback=?"在URL末尾添加一个额外字符以指定回调。"_=[TIMESTAMP]"除非将该cache选项设置为，否则通过在URL 后面附加查询字符串参数来禁用缓存true。
    • "text"：纯文本字符串。
    • 多个用空格分隔的值：从jQuery 1.5开始，jQuery可以将dataType从在Content-Type标头中接收到的dataType转换为所需的值。例如，如果要将文本响应视为XML，"text xml"则将其用于dataType。您也可以JSONP请求，把它收到的文本，并解释的jQuery为XML： "jsonp text xml"。类似地，诸如这样的速记字符串"jsonp xml"将首先尝试从jsonp转换为xml，如果失败，则将其从jsonp转换为文本，然后再从文本转换为xml。

  • global（默认值：true）

    类型：Boolean

    是否为此请求触发全局Ajax事件处理程序。默认值为true。设置为false防止像ajaxStart或这样的全局处理程序ajaxStop被触发。这可以用来控制各种[Ajax事件。

  • ifModified（default：false）

    类型：Boolean

    仅当响应自上一个请求以来已更改时，才允许请求成功。这是通过检查Last-Modified标头完成的。默认值为false，忽略标题。在jQuery 1.4中，此技术还会检查服务器指定的“ etag”以捕获未修改的数据。

  • isLocal（默认值：depends on current location protocol）

    类型：Boolean

    即使jQuery默认情况下也不会将当前环境识别为“本地”（例如文件系统）。下列协议目前公认的地方：file，*-extension，和widget。如果isLocal设置需要修改，建议在$.ajaxSetup()方法中进行一次。（添加的版本：1.5.1 (opens new window)）

  • jsonp

    类型：String或

    覆盖JSONP请求中的回调函数名称。将使用此值代替'callback =？'中的'callback'。网址中查询字符串的一部分。因此{jsonp:'onJSONPLoad'}会导致'onJSONPLoad=?'传递到服务器。从jQuery 1.5开始，设置jsonp选项以false防止jQuery将“？callback”字符串添加到URL或尝试使用“ =？” 进行转化。在这种情况下，您还应该显式设置jsonpCallback设置。例如，{ jsonp: false, jsonpCallback: "callbackName" }。如果您不信任Ajax请求的目标，出于安全考虑，请考虑将该jsonp属性设置false为。

  • jsonpCallback

    类型：String或Function()

    指定JSONP请求的回调函数名称。将使用此值代替jQuery自动生成的随机名称。最好让jQuery生成一个唯一的名称，因为它可以更轻松地管理请求并提供回调和错误处理。当您想启用更好的浏览器缓存GET请求时，可能需要指定回调。从jQuery 1.5开始，您还可以为该设置使用一个函数，在这种情况下，的值jsonpCallback设置为该函数的返回值。

  • mimeType

    类型：String

    覆盖XHR MIME类型的MIME类型。（添加的版本：1.5.1 (opens new window)）

  • password

    类型：String

    与XMLHttpRequest一起使用以响应HTTP访问认证请求的密码。

  • processData（默认值：true）

    类型：String

    默认情况下，data作为对象传递给选项的数据（技术上，不是字符串）将被处理并转换为查询字符串，以适合默认的内容类型“ application / x-www-form-urlencoded”。如果要发送DOMDocument或其他未处理的数据，请将此选项设置为false。

  • scriptAttrs

    类型：PlainObject

    定义具有附加属性的对象，该对象将在“脚本”或“ jsonp”请求中使用。键代表属性的名称，而值是属性的值。如果提供了此对象，它将强制使用脚本标签传输。例如，这可以用来设置nonce，integrity或crossorigin属性，以满足内容安全策略的要求。

  • scriptCharset

    类型：String

    仅在使用“脚本”传输时适用。charset在请求中使用的脚本标签上设置属性。当本地页面上的字符集与远程脚本上的字符集不同时使用。或者，charset可以scriptAttrs改为指定属性，这也将确保使用“脚本”传输。

  • StatusCode（默认值：{}）

    类型：PlainObject

    响应具有相应代码时将调用数字HTTP代码和函数的对象。例如，以下内容将在响应状态为404时发出警报：

    statusCode: {
    404: ()=>{
        alert('status code 404')
    },
    200: ()=> {
        alert('status code 200')
    }
}

    如果请求成功，则状态码函数采用与成功回调相同的参数；如果导致错误（包括3xx重定向），则它们将使用与error回调相同的参数。

  • tranitional

    类型：Boolean

    true如果您希望使用传统的参数序列化 (opens new window)样式，请将其设置为。

  • URL（默认值：The current page）

    类型：String

    包含请求发送到的URL的字符串。

  • usernmae

    类型：String

    XMLHttpRequest用来响应HTTP访问身份验证请求的用户名。

  • xhr（默认值：ActiveXObject when available (IE), the XMLHttpRequest otherwise）

    类型：Function()

    用于创建XMLHttpRequest对象的回调。可用时默认为ActiveXObject（IE），否则默认为XMLHttpRequest。重写以提供您自己的XMLHttpRequest实现或工厂增强功能。

  • xhrFields

    类型：PlainObject

    要在原始XHR对象上设置的fieldName-fieldValue对对象。例如，你可以用它来设置withCredentials到true如果需要跨域请求。

    $.ajax({
   url: a_cross_domain_url,
   xhrFields: {
      withCredentials: true
   }
});

该$.ajax()函数是jQuery发送的所有Ajax请求的基础。通常没有必要直接调用此函数，因为有几个更高级别的替代项（例如$.get()和）.load()可用，并且更易于使用。但是，如果需要的选项较少，$.ajax()则可以更灵活地使用。

最简单的说，该$.ajax()函数可以不带任何参数来调用：

$.ajax();

**注意：**可以使用该$.ajaxSetup()功能全局设置默认设置。

本示例，不使用任何选项，将加载当前页面的内容，但对结果不执行任何操作。要使用结果，可以实现回调函数之一。
