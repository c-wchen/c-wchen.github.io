不同体系的CPU采用的字节序列是不一样的，主要分为两种大端（Big-Endian）小端（Little-Endian）

# 大小端

大端模式： 是指数据的高字节保存在低地址位

小端模式： 是指数据的低字节保存在高地址位

注意： 是字节（Byte）序列不同，而非比特（Bit）的区别

如： int a = 0x12345678

# 为什么需要大小端，直接统一不行吗？

计 算机电路先处理低位字节，效率比较高，因为计算都是从低位开始的。所以，计算机的内部处理都是小端字节序。在计算机内部，小端序被广泛应用于现代 CPU 内部存储数据；而在其他场景，比如网络传输和文件存储则使用大端序。

# 大小端转换

针对32字节进行转换，当然也可以扩展

static inline unsigned int convert_endian(unsigned int data) {
    return ((0xFF & data) << 24) | ((0XFF << 8 & data) << 8) |
        ((0XFF << 16 & data) >> 8) | ((0XFF << 24 & data) >> 24);
}

如何检查大小端

static inline bool is_big_endian() {
    unsigned short data = 0x1234;
    return *(char *)&data == 0x12;
}

# 结构体和位域布局

结构体布局（未采取大小端方式进行存储）：

结构体（联合体）的成员所占内存地址依次增高，第一个成员位于低地址处，最后一个成员位于高地址处，但是结构体成员的内存并不是连续的，编译器会对成员做上述“对齐”处理。

位域布局：

struct Test{
    unsigned short a:5;
    unsigned short b:5;
    unsigned short c:6; 
}; 

int main(int argc, char**){
    Test t;
    t.a=16;
    t.b=4;
    t.c=0;
    return 0;
}

大端模式： a-->b-->c

小端模式： c-->b-->a

# 领域应用

网络字节序：TCP/IP各层协议将字节序定义为Big-Endian，因此TCP/IP协议中使用的字节序通常称之为网络字节序。
